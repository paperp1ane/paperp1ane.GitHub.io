2021-03-20 6 min read

Common JS

1.创建webpack demo，具体步骤不阐述
2.在根目录建立src文件夹
我们创建了两个文件
index.js

    const say = require('./hi')

和 hi.js

    module.exports = function hi(value) {
        alert('say'+ value)
    }

3.执行编译命令。

node_modules/.bin/webpack --mode development

没有加入Babel，采用common.js规范
打开dist文件夹，打开编译后的文件

可以看到整个代码并不长，精简一下如下


    (() => {
        <!-- 业务模块 -->
        __webpack_modules__ = {
            [路径作为moduleID]: [处理后的脚本]  
        )
        <!-- 缓存模块 -->
        __webpack_module_cache__ = {

        }
        <!-- webpack require -->
        function __webpack_require__(moduleID) {

        }
        <!-- 入口文件 -->
        var __webpack_exports__ = __webpack_require__("./src/index.js");
    })()

1.可以看到，webpack重写了一个webpack_require方法去加载模块，实际上符合commonjs的规则。
2.通过入口文件indexjs去启动加载模块
3.在webpack_require方法中，会先判断当前路径[moduleID]模块是否已经在缓存中，如果有，直接返回缓存的结果。如果没有会创建一个新的module = { exports: {} }对象，同时将该对象放到webpack__module_cache中。下一步就是执行moduleID对应的function，这个function会有三个参数，如果业务代码中有commonjs的require,则webpack的__webpack_require__方法会作为参数传入对应的function中执行。最后返回模块的exports对象

查看编译后的文件
在声明webpack_require函数之后有扩展了新的函数。
1.webpack_require.r
如果symbol可用的话，回味当前module添加一个自定义类型标签，标记为Module
同时在module.exports上定义__esModule属性，表明为一个模块对象
2.webpack_require.o
hasOwnProperty的简写
3.webpack_require.d
为了和ES6的特性保持一致，将模块对应的function添加到module.exports对象上，
重写了getter 函数，没有设置属性setter函数。防止修改module.exports对应的function。
